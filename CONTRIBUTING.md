# Contributing to AIGMOS

## Version

v0.1

## Status

Draft

## Purpose

Define how future contributors should approach design, documentation, and implementation work in AIGMOS.

## Initial Contribution Principles

- Keep changes focused and easy to review.
- Update documentation when a decision changes.
- Avoid adding frameworks, dependencies, or generated code without a clear project decision.
- Mark uncertain ideas as draft or open questions.
- Preserve separation between product, game design, AI, architecture, data, UX, and development notes.
