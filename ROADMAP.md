# AIGMOS Roadmap

## Version

v0.1

## Status

Draft

## Purpose

This roadmap outlines the planned project phases for AIGMOS. Details are intentionally high level during the design phase.

## Phase 0: Design Foundation

Status: Planned

Purpose: Establish the constitution, product vision, core gameplay loop, documentation structure, and initial design vocabulary.

To be defined:

- Design constitution
- Product vision
- Core gameplay loop
- Initial system boundaries
- Documentation standards

## Phase 1: Text Prototype

Status: Planned

Purpose: Validate the core AI Game Master experience through a text-first prototype.

To be defined:

- Minimal interaction loop
- Session state model
- Basic adventure flow
- AI response expectations

## Phase 2: Web Demo

Status: Planned

Purpose: Create a lightweight browser-based demo once the text prototype proves the core loop.

To be defined:

- Web UX scope
- Session view
- Character and campaign display
- Demo constraints

## Phase 3: Mobile App Prototype

Status: Planned

Purpose: Explore mobile interaction patterns for solo and multiplayer play.

To be defined:

- Mobile navigation model
- Campaign continuity experience
- Notification and session-resume assumptions
- Platform constraints

## Phase 4: Closed Alpha

Status: Planned

Purpose: Test the broader AIGMOS experience with a limited group of players.

To be defined:

- Alpha success criteria
- Feedback process
- Safety and moderation expectations
- Operational requirements
